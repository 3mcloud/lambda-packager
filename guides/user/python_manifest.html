Python Lambda Packager Manifest File¶

Usage¶

To allow for the packager to package several different lambdas at once, we leverage a manifest file. The name of the file is unimportant, but the schema of the file is defined below. The file should be a .yml, .yaml, or .json file.

Spec¶

1kind: lambda-packager
2version: # The version of the spec to use. Currently only `1.0.0`.
3lambdas: # List of lambdas to package
4  - name: add_to_cell # Build name, only used to keep builds independent. Must be unique
5    environment_overrides:
6    # Any environment variable supported with the Python Packager documentation

Example¶

 1---
 2kind: lambda-packager
 3version: 1.0.0 # Manifest file version
 4lambdas:
 5  - name: add_to_cell # This is just used for building, it does not effect the artifact file.
 6    environment_overrides:
 7      LAMBDA_CODE_DIR: ./lambdas/mock_processor_objects/add_to_cell
 8      REQUIREMENTS_FILE: ./requirements.txt
 9      ARTIFACT_NAME: po_add_to_cell.zip
10      FAIL_ON_TOO_BIG: True # Boolean or String is fine.
11      SSH_FLIP: True
12  - name: extract_cell
13    environment_overrides:
14      LAMBDA_CODE_DIR: ./lambdas/mock_processor_objects/extract_cell
15      REQUIREMENTS_FILE: ./requirements.txt
16      ARTIFACT_NAME: po_extract_cell.zip
17      FAIL_ON_TOO_BIG: True
18      SSH_FLIP: True
19  - name: mock_file_writer
20    environment_overrides:
21      LAMBDA_CODE_DIR: ./lambdas/mock_file_writer
22      REQUIREMENTS_FILE: ./requirements.txt
23      ARTIFACT_NAME: mock_file_writer.zip
24      FAIL_ON_TOO_BIG: True
25      SSH_FLIP: True
